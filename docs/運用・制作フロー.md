# ごみニコ 運用・制作フロー

> 対象：宮澤さん一人法人 / AI多用 / 1日15〜30分運用
> 最終更新: 2026-07-16（Firebase前提の旧記述をCloudflare Pages/GitHubの現行構成に更新）

---

## 全体フロー概観

```
自治体選定 → データ収集 → JSON作成 → 実装 → GitHubへpush → Cloudflare Pagesに自動デプロイ
    ↑                                                              ↓
    └──────────────── トラフィック実績 → 自治体営業 ────────────────┘
```

フェーズは2段階。

- **Phase 1（ゲリラ期）** 自分でサイトを運営し、アクセスを稼いで実績を作る ← いまここ
- **Phase 2（SaaS期）** 実績を武器に自治体と契約、月額収益へ移行

**含意**: 当初「候補: 和光市・新座市・朝霞市」としていた3市も、現在はすべてアプリ・LINE導入済みと判明（ドキュメント作成時点から状況が変化した可能性）。都心・さいたま近郊の人口規模の大きい自治体は、ほぼアプリ導入済みとみてよい状況。**「PDF・Webのみでアプリ未導入」という当初の条件に完全一致する近隣自治体は、実質的に戸田市（公式ごみ分別アプリ・LINE Bot共に無し。ただし粗大ごみ申込のみLINE対応・市公式の汎用AIチャットボットあり）程度に絞られる**。この「未導入の空白地帯」は今後さらに縮小していく可能性が高いため、条件に合う自治体が見つかった際は先送りせず早めに着手する方が有利と考えられる。より完全に競合のない自治体を求めるなら、さいたま近郊から離れた地域まで探索範囲を広げる必要があるが、その場合は「将来の営業訪問を想定」という当初の狙いとトレードオフになる。

**2026-07-23追記：Geminiが作成した営業先候補リストの裏取り結果**。太平さんがGeminiに依頼した市場調査（「引き算アプローチ」：総務省1,788自治体マスターから大手競合の導入自治体を除外し、残りを人口順に精査する手法）を共有いただき、記載candidateを一次資料で検証した。

- **手法自体は有効**：「大手アプリ導入自治体を除外して残りを絞り込む」という引き算アプローチ、および「ごみ処理組合（蕨戸田衛生センター組合等）単位でセットアプローチする」という発想はどちらも合理的で、今後の候補地選定に採用する価値がある
- **候補地の記載には誤りが複数あった（要注意）**:
  - 三郷市: Geminiは「有力」としたが、市公式アプリ「ポケットみさと」でごみ情報を提供済み。除外対象
  - 鴻巣市: Geminiは候補としたが、「ごみスケ」（econosu）を導入済み。Gemini自身が「ごみスケ等を除外」としている手法と矛盾しており、実行時に取りこぼしがあった可能性
  - 我孫子市: Geminiは「HP上のPDF・テキスト案内が中心」としたが、実際は公式LINEアカウントで地区登録・前日/当日通知・トーク画面でのごみ品名検索ができる、実質的なチャットボット運用済み。除外対象
  - 小金井市: Geminiは「独自の厳しい分別ポリシーのため大手汎用アプリで対応しきれずWeb中心」としたが、実際は小金井市独自の専用ごみ分別アプリ（カレンダー・アラート・分別辞典・FAQ・販売店MAP機能付き）を配信済み。除外対象
- **裏取りできた候補**:
  - **蕨市**（人口約7.6万人・戸田市と同じ蕨戸田衛生センター組合）: 公式サイト（2026年4月2日更新の最新ページ）を確認したところ、PDF（分別表・多言語版）のみでの案内、アプリ・LINE bot共になし。裏取り済みの有力候補
  - **桶川市**: 独自アプリ「おけプッシュ」は運用終了済み。現在はキーワード・50音検索ができるWebページ＋PDFでの案内が中心（単純なPDFよりは進んでいるが、専用アプリ・LINE botは確認できず）。準有力候補
- **未確認（要追加調査）**: 八千代市（情報得られず）、武蔵野市（独自アプリの情報は見当たらなかったが確証はまだ薄い）

**教訓**: AIによる市場調査は、大手アプリベンダーの「導入自治体一覧」のような一次資料に基づかない場合、検索結果の要約だけでは誤りが混入しやすい（今回はGemini提示候補6件中3件で誤りを確認）。営業リストとして実際に自治体へ接触する前には、必ず各自治体の公式サイトで最終確認すること。

## STEP 6｜デプロイ（Cloudflare Pages / GitHub）

```
ローカルで編集 → git commit → git push → Cloudflare Pagesが自動ビルド・公開（1〜2分）
```

- 本番URL: `https://gomi-nico.jp/{city}/`（例: /shiki/）
- テスト: main以外のブランチにpushすると `<ハッシュ>.gomi-nico.pages.dev` のプレビューURLが自動発行される
- 新自治体追加時: `{city}/index.html`・`data_{city}.json`・sitemap.xml・トップのカルーセルを更新（詳細は handoff-2026-07-16.md）
