# ごみ分別SaaSプラットフォーム 運用・制作フロー

> 対象：宮澤さん一人法人 / AI多用 / 1日15〜30分運用

---

## 全体フロー概観

```
自治体選定 → データ収集 → JSON作成 → デザイン → 実装 → Firebase公開
    ↑                                                        ↓
    └──────────────── トラフィック実績 → 自治体営業 ──────────┘
```

フェーズは2段階です。

- **Phase 1（ゲリラ期）** 自分でサイトを立ち上げ、アクセスを稼いで実績を作る
- **Phase 2（SaaS期）** 実績を武器に自治体と契約、月額収益へ移行

---

## 各ステップ詳細

---

### STEP 1｜自治体の選定

**担当：自分（AIが補助）**

#### 選定基準（優先順）

| 条件 | 理由 |
|------|------|
| 人口 3〜10万人 | 大きすぎず、競合が少ない |
| 既存ごみアプリなし | さんあ〜る等が未導入の自治体を狙う |
| PDFまたはWebのみでアプリ未対応 | データ取得しやすい |
| 埼玉・東京近郊 | 将来の営業訪問を想定 |

#### 選定の進め方

1. Claudeに「埼玉県の市区町村でさんあ〜るに登録されていない自治体リストを調べて」と依頼
2. 人口・競合状況をスプレッドシートに整理
3. 自分で最終選定

---

### STEP 2｜データ収集

**担当：自分（AIが補助）**

#### 収集方法

対象自治体のウェブサイト・PDFから以下を集める。

- ごみ収集カレンダー（地区別・曜日・週）
- 分別品目一覧（50〜100品目）
- 粗大ごみ・危険ごみのルール
- 問い合わせ先

#### ツール

- **ClaudeにPDFをアップロード** → 「このPDFから収集スケジュールをJSON形式で抽出して」と依頼
- **Claude in Chrome** → ウェブページを直接読み込んでデータ抽出

---

### STEP 3｜データ管理（Spreadsheet → JSON）

**← ここが一番の検討ポイントです**

#### 推奨フロー（2段階）

**Phase 1：Claudeに直接JSON生成させる（最もシンプル）**

```
PDF・Webページ → Claude → data_城市名.json → Firebase
```

- Claudeに「このデータを data_shiki.json と同じ形式のJSONにして」と依頼するだけ
- スプレッドシート不要、追加ツール不要
- 更新のたびにClaudeに依頼して再生成

**Phase 2：Google Sheets → 自動JSON変換（自治体に自己管理させる場合）**

```
自治体担当者がSheets入力 → Apps Script → Firebase自動デプロイ
```

自治体に「このスプレッドシートに入力してください」と渡すだけで、
Apps Scriptが自動でJSONに変換してFirebaseに反映できます。

#### Q. 直接Sheetsから読み込む方法は？

スクリプトからGoogle Sheets APIを直接叩く方法もありますが、**推奨しません**。
APIキーの管理、CORS問題、レスポンス速度など運用コストが上がります。
「Sheets → JSON → Firebase静的配信」の方がシンプルで速いです。

---

### STEP 4｜デザイン

**担当：自分（FigmaでUI確認）**

#### 流れ

1. 新機能や変更点を **Figmaでモックアップ**
2. 全体デザインは既存の `index.html` のデザイントークンを流用
3. 自治体ごとのブランドカラーは `data_*.json` の `brand_color` を変えるだけ

#### 自治体カスタマイズ可能な要素（JSON側で制御）

```json
{
  "name": "川越市",
  "brand_color": "#8B2E2E",
  "ad_type": "official"
}
```

基本的にデザイン変更なしで複数自治体に対応できます。

---

### STEP 5｜実装

**担当：Claude（自分はチェックのみ）**

#### 依頼の仕方

- 新機能：「〇〇機能を script.js に追加して」
- バグ修正：「〇〇が動かない。index.htmlとscript.jsを見て直して」
- 新自治体追加：「このデータを data_kawagoe.json の形式で作って」

#### VSCode でのチェック方法

1. Live Server で `http://127.0.0.1:5500/?city=shiki` を開く
2. Chrome DevTools（F12）のコンソールでエラー確認
3. スマホ表示はDevToolsの「デバイスツールバー（Ctrl+Shift+M）」で確認

---

### STEP 6｜Firebase 構成・デプロイ

**担当：Claude が手順書を作成、自分が実行**

#### ファイル構成（Firebase Hosting）

```
Firebase Hosting
├── index.html       ← 全自治体共通テンプレート
├── script.js        ← 全自治体共通ロジック
├── data_shiki.json       ← 志木市
├── data_kawagoe.json     ← 川越市（将来）
└── data_tokorozawa.json  ← 所沢市（将来）
```

#### URL設計（2パターン）

**パターンA：サブドメイン（推奨）**
```
shiki.gomi-navi.jp     → Firebase rewrites → /?city=shiki
kawagoe.gomi-navi.jp   → Firebase rewrites → /?city=kawagoe
```
自治体ごとに独立したURLに見える。営業しやすい。

**パターンB：パスパラメータ**
```
gomi-navi.jp/?city=shiki
gomi-navi.jp/?city=kawagoe
```
設定がシンプル。Phase 1はこちらで十分。

#### デプロイの流れ

```bash
# 初回のみ
npm install -g firebase-tools
firebase login
firebase init hosting

# 毎回のデプロイ（JSONを追加・更新したとき）
firebase deploy
```

Firebase CLIのインストール・初期設定はClaudeに手順書を作ってもらえます。

### STEP 7｜収益化の設定

#### Google AdSense

1. [adsense.google.com](https://adsense.google.com) でアカウント作成
2. サイト審査（Firebase のURLを登録）→ 通常1〜2週間
3. 審査通過後、`index.html` のコメントアウトを外す
4. `data_*.json` の `"ad_type": "affiliate"` のまま → AdSenseが表示される

**注意：** 審査にはある程度のコンテンツと流入が必要。先にFirebase公開してSEOを育ててから申請する。

#### 楽天アフィリエイト

1. [affiliate.rakuten.co.jp](https://affiliate.rakuten.co.jp) でアカウント作成
2. `data_shiki.json` の `affiliate_items[].url` に楽天アフィリエイトURLを設定
3. クリック → 購入で報酬発生（目安：購入額の1〜3%）

**高CPMが狙えるカテゴリ（引越し・不用品回収）：**
- 引越し一括見積もり（A8.net / バリューコマース）→ 1件1,000〜3,000円
- 不用品回収の見積もり → 1件800〜2,000円
- これらは `affiliate_items` に追加するだけで表示切替可能

#### Google Analytics 4

`index.html` の `<head>` に設置するだけ。

```html
<!-- Google Analytics -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>
```

#### Looker Studio（レポート）

- Google Analytics のデータを自動でグラフ化
- 「どの自治体ページのアクセスが多いか」「どの検索ワードが多いか」が一目でわかる
- 自治体への営業資料にそのまま使える
- 設定はAnalytics連携後5分で完了
