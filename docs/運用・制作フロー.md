# ごみニコ 運用・制作フロー

> 対象：宮澤さん一人法人 / AI多用 / 1日15〜30分運用
> 最終更新: 2026-07-16（Firebase前提の旧記述をCloudflare Pages/GitHubの現行構成に更新）

---

## 全体フロー概観

```
自治体選定 → データ収集 → JSON作成 → 実装 → GitHubへpush → Cloudflare Pagesに自動デプロイ
    ↑                                                              ↓
    └──────────────── トラフィック実績 → 自治体営業 ────────────────┘
```

フェーズは2段階。

- **Phase 1（ゲリラ期）** 自分でサイトを運営し、アクセスを稼いで実績を作る ← いまここ
- **Phase 2（SaaS期）** 実績を武器に自治体と契約、月額収益へ移行

---

## STEP 1｜自治体の選定

**担当：自分（AIが補助）**

| 条件 | 理由 |
|------|------|
| 人口 3〜10万人 | 大きすぎず、競合が少ない |
| 既存ごみアプリなし | さんあ〜る等が未導入の自治体を狙う |
| PDFまたはWebのみでアプリ未対応 | データ取得しやすい |
| 埼玉・東京近郊 | 将来の営業訪問を想定 |

候補: 和光市・新座市・朝霞市

## STEP 2｜データ収集

対象自治体のサイト・PDFから収集するもの: 収集カレンダー（地区別）／分別品目一覧／粗大ごみ・危険ごみのルール／問い合わせ先。

- ClaudeにPDFをアップロード →「収集スケジュールをJSON形式で抽出して」と依頼

## STEP 3｜データ作成（Claudeで直接JSON生成）

```
PDF・Webページ → Claude → data_{city}.json → GitHubへpush
```

- 「このデータを data_shiki.json と同じ形式のJSONにして」と依頼するだけ
- **必須: 収集曜日は一次資料と全数照合すること**（志木市で館2-2-4地区の曜日誤りが混入した教訓。細分化された例外地区に注意）
- スプレッドシート連携の自動化は自治体数が増えてから検討（ロードマップ フェーズ4）

---

## STEP 4｜デザイン

- 既存のデザイントークンを流用（詳細は `UIデザイン仕様.md`）
- 自治体ごとの差分は `data_{city}.json` の `brand_color` 等で制御。デザイン変更なしで横展開可能

## STEP 6｜デプロイ（Cloudflare Pages / GitHub）

```
ローカルで編集 → git commit → git push → Cloudflare Pagesが自動ビルド・公開（1〜2分）
```

- 本番URL: `https://gomi-nico.jp/{city}/`（例: /shiki/）
- テスト: main以外のブランチにpushすると `<ハッシュ>.gomi-nico.pages.dev` のプレビューURLが自動発行される
- 新自治体追加時: `{city}/index.html`・`data_{city}.json`・sitemap.xml・トップのカルーセルを更新（詳細は handoff-2026-07-16.md）

## STEP 7｜収益化

### Google AdSense
1. ビジネスアカウントで申請（gomi-nico.jp）
2. 審査通過後、各ページのダミー広告枠に本番コードを差し込み + ads.txt設置

### 楽天アフィリエイト（法人）
- 法人フォーム経由で申請済み（2026-07-16）。承認後、`CAT_PRODUCTS`（script.js）とお得情報タブに正規リンクを設定
- 高単価領域（引越し一括見積・不用品回収）はA8.net / バリューコマース等を承認後に検討

### 計測
- GTM（GTM-MNH8H4L2）→ GA4 + Microsoft Clarity 導入済み。GA4は内部トラフィック除外設定済み
- Looker Studio はデータが溜まってから（営業資料に転用）
