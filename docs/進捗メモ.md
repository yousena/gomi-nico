## プロジェクト概要

---

## URL構造

```
https://gomi-nico.jp/           ← トップページ（landing page）
https://gomi-nico.jp/shiki/     ← 志木市アプリ
https://gomi-nico.jp/about.html
https://gomi-nico.jp/privacy.html
https://gomi-nico.jp/ads.html
```

- 自治体ページは `/自治体コード/` 形式（`?city=` クエリパラメータは廃止）
- `shiki/index.html` に `<base href="/">` と `<meta name="city" content="shiki">` を設置

## 技術スタック

- HTML / Tailwind CSS (Play CDN) / Vanilla JavaScript
- Material Symbols Rounded（カテゴリアイコン）、Material Symbols Outlined（ナビアイコン）
- データはJSON形式 `/data_{city}.json`
- AdSense枠はダミー表示（本番時にコードを差し込む）

---

## 主要機能（アプリ側 shiki/index.html ＋ script.js）

1. **検索ボックス** - ごみ名のサジェスト検索、分別・出し方・注意点を表示
2. **丁目選択** - LocalStorage に保存（再訪時も保持）
3. **今日のごみ** - 選択地区の当日収集を表示
4. **明日のごみ** - 翌日収集を表示
5. **カレンダー** - 月次カレンダーでごみ収集日一覧（アイコン22px・複数表示対応）

---

## マルチテナント対応（他自治体追加方法）

1. `data_{city}.json` を作成（志木市JSONを複製・編集）
2. `{city}/index.html` を作成（shiki/index.html を複製、meta[name="city"]とcanonicalを変更）
3. `sitemap.xml` にURLを追加
4. `index.html` トップページの自治体カードに追加

### data_{city}.json の拡張フィールド（マルチテナント用）
```json
"official_url": "自治体公式URL",
"official_url_label": "リンクテキスト",
"citation_date": "2026年7月",
"faq": [ { "q": "...", "a": "..." } ]
```
